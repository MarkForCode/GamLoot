# Skill: Bug 修復

## 觸發條件
用戶說「壞了」、「不能用了」、「有錯誤」、「掛點」、「bug」

## 工作流程

### 1. 收集資訊
- [ ] 請用戶描述錯誤訊息
- [ ] 詢問發生情境 (什麼操作觸發)
- [ ] 確認環境 (dev/prod, 瀏覽器/手機)
- [ ] 取得 logs 或截圖

### 2. 重現問題
- [ ] 本地重现 bug
- [ ] 檢查相關 logs
- [ ] 確認觸發條件

### 3. 分析問題
- [ ] 找到 root cause
- [ ] 確認是前端/後端/資料庫問題
- [ ] 找出相關程式碼位置

### 4. 修復
- [ ] 撰寫修復程式碼
- [ ] 不要引入新 bug
- [ ] 保持現有功能正常

### 5. 驗證
- [ ] 本地測試通過
- [ ] lint/typecheck/build 通過
- [ ] 確認修復成功

### 6. 完成回報
- [ ] 說明修復內容
- [ ] 說明 root cause (如有必要)
- [ ] 請用戶測試

## 修復原則
- 不要只修表面的錯誤，要找根本原因
- 如果不確定，請先問清楚
- 小心不要製造新的 bug
- 複雜的 bug 需要先規劃再動手

## 緊急修復 (Hotfix)
如果用戶說「很緊急」、「系統掛了」:
1. 立即嘗試重現問題
2. 快速找到可行解
3. 先上線再說 (如果無法立即修復)
4. 後續再優化

## 回報格式
```
## Bug 修復報告

**問題**: [簡述]
**原因**: [root cause]
**修復**: [做了什麼]
**驗證**: [測試結果]
```
