# Skill: 需求分析

## 觸發條件
用戶說「要做OO功能」、「想要有XX」、「新增一個OO系統」

## 工作流程

### 1. 理解需求
- [ ] 詢問清楚功能範圍 (scope)
- [ ] 了解目標使用者 (user)
- [ ] 確認商業價值 (why now)
- [ ] 找出相關的現有功能 (existing)

### 2. 技術評估
- [ ] 檢查現有架構是否支援
- [ ] 需要哪些新元件 (database, api, ui)
- [ ] 評估複雜度 (簡單/中等/複雜)
- [ ] 估算時間

### 3. 列出需求清單
```
## 功能名稱

### Must Have
- [ ] 需求項目 1
- [ ] 需求項目 2

### Nice to Have
- [ ] 優化項目 1
- [ ] 優化項目 2

### Out of Scope
- 這次不做的事項
```

### 4. 技術規格 (如需要)
- [ ] 資料庫 schema 變更
- [ ] API endpoint 設計
- [ ] 前端 component 規劃

### 5. 確認
- [ ] 再次與用戶確認理解正確
- [ ] 確認優先順序
- [ ] 開始實作或等待用戶確認

## 範例輸出

```
## 需求分析總結

**功能**: 用戶登入系統
**優先**: 高
**複雜**: 中等

### 技術需求
- 資料表: users (已存在)
- API: POST /auth/login
- Frontend: 登入頁面 + 認證狀態管理

### 下一步
[等待確認] 或 [開始實作]
```

## 重要提醒
- 不要假設需求，沒聽懂就問
- 先問清楚再做，避免做錯重來
- 複雜功能才需要詳細規劃
